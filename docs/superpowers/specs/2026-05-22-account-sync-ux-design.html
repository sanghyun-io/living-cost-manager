계정 및 동기화 UX 고도화 설계

공개 API 배포 이후, 사용자가 로컬 저장과 서버 동기화의 차이를 명확히 이해하고 안전하게 데이터를 옮길 수 있게 만드는 1차 UX 고도화 설계입니다.

목표

현재 앱은 localStorage 기반 사용 흐름과 서버 계정 기반 동기화 흐름을 함께 제공합니다. 이번 고도화의 목표는 사용자가 어느 저장 모드에 있는지, 서버와 언제 동기화됐는지, 실패했을 때 무엇을 해야 하는지 한눈에 알 수 있게 만드는 것입니다.

  • 로컬 모드와 서버 계정 모드를 명확히 분리합니다.
  • 로컬 모드의 제한과 데이터 손실 위험을 명확히 경고합니다.
  • 서버 연결 상태와 마지막 동기화 시각을 표시합니다.
  • 로컬 데이터와 서버 데이터가 충돌할 때 선택지를 명확히 제공합니다.
  • 동기화 실패가 데이터 손실로 보이지 않도록 로컬 저장 유지 상태를 안내합니다.
  • 공유/멤버 관리는 동기화 설정 안에서 자연스럽게 접근하게 합니다.

비목표

  • 새 분석 리포트 추가
  • 고정비 데이터 모델 변경
  • 실시간 공동 편집
  • 비밀번호 재설정 이메일
  • Cloudflare Pages 이전 작업

접근안

추천상태 중심 동기화 패널

기존 데이터 관리 영역 안에 계정/동기화 패널을 정리하되, 상태 표시를 가장 위에 둡니다. 로그인 폼보다 현재 상태, 마지막 동기화, 다음 액션을 먼저 보여줘 사용자가 판단하기 쉽게 합니다.

온보딩 위저드

서버 계정 사용을 처음 시작할 때 단계형 마법사를 제공하는 방식입니다. 이해는 쉽지만 현재 앱 규모에 비해 화면 전환이 커지고, 반복 사용 시 번거로울 수 있습니다.

자동 동기화 우선

로그인만 하면 자동으로 서버 데이터를 덮어쓰거나 업로드하는 방식입니다. 빠르지만 로컬 데이터가 이미 있는 사용자의 데이터 선택권이 약해져 위험합니다.

선택 설계

상태 중심 동기화 패널을 사용합니다. 핵심은 “지금 어디에 저장되고 있는가”와 “다음에 무엇을 누르면 되는가”를 명확히 보여주는 것입니다. 

데이터 관리
  계정 및 동기화
    상태 카드: 서버 연결됨 / 연결 실패 / 로컬 전용
    세션 카드: 사용자, 워크스페이스, 권한
    동기화 카드: 마지막 동기화, 원격 데이터 상태, 다음 액션
    충돌 선택: 서버 불러오기 / 이 브라우저 업로드 / 로컬 전용 유지
  공유 관리
    멤버 목록
    초대 목록
    소유자 전용 관리 액션

상태 모델

프론트는 동기화 상태를 명시적인 상태값으로 계산합니다.

  • local-only: API URL이 없거나 사용자가 서버에 로그인하지 않았습니다.
  • server-available: API URL이 있고 로그인 전입니다.
  • signed-in: 서버 세션이 있고 워크스페이스가 선택됐습니다.
  • checking: 서버 snapshot 또는 세션 복원 확인 중입니다.
  • needs-decision: 로컬 데이터와 서버 데이터 중 무엇을 기준으로 할지 선택해야 합니다.
  • syncing: 업로드 또는 다운로드 중입니다.
  • synced: 마지막 동기화가 성공했습니다.
  • failed: 서버 요청이 실패했습니다. 로컬 저장은 계속 유지됩니다.
  • auth-expired: 토큰이 만료되었거나 권한이 거부되어 다시 로그인이 필요합니다.

로컬 모드 제약 및 경고

로컬 모드는 계속 지원하지만, 서버 동기화의 대체가 아니라 기기 안에만 저장되는 제한 모드로 안내합니다. 사용자가 서버 계정을 쓰지 않아도 앱을 사용할 수 있어야 하지만, 데이터가 브라우저 저장소에만 있다는 사실은 반복해서 분명히 보여줘야 합니다.

반드시 명시할 제약

  • 다른 기기나 다른 브라우저와 자동 동기화되지 않습니다.
  • 브라우저 데이터 삭제, 캐시 정리, 시크릿 모드 종료, 기기 교체 시 데이터가 사라질 수 있습니다.
  • 로그인, 워크스페이스 공유, 멤버 관리, 서버 백업 기능을 사용할 수 없습니다.
  • 브라우저 저장소 용량이나 정책에 영향을 받을 수 있습니다.
  • 데이터 복구는 사용자가 직접 Export해둔 백업 파일이 있을 때만 가능합니다.

경고 표시 위치

  • 첫 진입 또는 로컬 사용자 선택 화면: 로컬 모드는 이 브라우저에만 저장된다는 경고를 표시합니다.
  • 데이터 관리 영역: 로컬 전용 상태 badge와 Export 권장 문구를 표시합니다.
  • 전체 Export 영역: 서버 계정이 없으면 “기기를 바꾸기 전 반드시 백업” 문구를 표시합니다.
  • 서버 연결 실패 상태: “로컬 저장은 유지되지만 서버 백업은 되지 않음”을 표시합니다.

권장 문구

로컬 모드: 이 브라우저에만 저장됩니다.
브라우저 데이터를 삭제하거나 기기를 바꾸면 복구할 수 없습니다.
계속 로컬 모드를 쓸 경우 정기적으로 전체 Export 백업을 보관하세요.

사용자 흐름

1. 로컬 사용자

  1. 앱을 열면 기존 localStorage 데이터는 그대로 표시됩니다.
  2. 데이터 관리 영역에서 “로컬 전용” 상태와 제약 경고를 봅니다.
  3. 서버 계정 연결 가능 상태와 전체 Export 백업 권장을 함께 봅니다.
  4. 서버를 쓰지 않아도 기존 기능은 계속 사용할 수 있습니다.

2. 서버 가입 또는 로그인

  1. 이메일, 비밀번호, 이름으로 가입하거나 로그인합니다.
  2. 로그인 성공 후 워크스페이스를 자동 선택합니다.
  3. 서버 snapshot을 확인하고 로컬 데이터 상태와 비교합니다.

3. 로컬 데이터가 있고 서버가 비어 있음

  1. “이 브라우저 데이터를 서버에 업로드”를 주 액션으로 보여줍니다.
  2. 사용자가 업로드하면 서버 snapshot을 저장하고 마지막 동기화 시각을 갱신합니다.

4. 서버 데이터가 있고 로컬 데이터도 있음

  1. 자동 덮어쓰기를 하지 않습니다.
  2. 서버 데이터 불러오기, 이 브라우저 데이터 업로드, 로컬 전용 유지 중 하나를 선택하게 합니다.
  3. 선택 전에는 서버/로컬의 월 수입, 항목 수, 총 월 환산액을 비교 표시합니다.

5. 서버 요청 실패

  1. 편집 내용은 localStorage에 유지합니다.
  2. 상태를 “서버 동기화 실패, 로컬 저장됨”으로 표시합니다.
  3. 사용자가 “다시 시도”를 누를 수 있게 합니다.

화면 변경

  • 상단 요약 영역에는 서버 상태를 과하게 노출하지 않습니다. 주 화면은 생활비 관리에 집중합니다.
  • 데이터 관리 모달 또는 패널 안에 “계정 및 동기화” 섹션을 둡니다.
  • 로컬 전용 상태일 때는 dismissible 안내가 아니라 persistent warning으로 제약을 표시합니다.
  • 상태 badge는 색상과 텍스트를 함께 사용합니다. 색상만으로 상태를 전달하지 않습니다.
  • 버튼은 성격별로 분리합니다: 계정 액션, 동기화 액션, 공유 관리 액션.
  • 위험한 액션은 보조 버튼으로 두고, 삭제처럼 되돌리기 어려운 액션만 confirm을 사용합니다.

데이터 표시

동기화 결정 전에 서버와 로컬 데이터를 비교할 때 다음 정보를 표시합니다.

  • 월 수입
  • 고정비 항목 수
  • 월 환산 고정비 합계
  • 카테고리 수
  • 카드 수
  • 서버 snapshot 생성 또는 갱신 시각이 있으면 표시

에러 처리

  • 401 또는 403은 세션 만료/권한 문제로 분리해 “다시 로그인 필요”로 안내합니다.
  • 네트워크 오류는 “서버 연결 실패”로 표시하고 로컬 저장이 유지된다는 문구를 함께 보여줍니다.
  • 서버 validation 오류는 API message를 그대로 길게 노출하지 않고 짧은 안내문으로 정리합니다.
  • 동기화 실패 시 사용자가 입력한 최신 로컬 데이터는 절대 삭제하지 않습니다.

테스트

  • API base URL이 없으면 local-only 상태가 계산되는지 테스트합니다.
  • local-only 상태에서는 로컬 모드 제한 경고와 전체 Export 권장이 표시되는지 테스트합니다.
  • 로그인 성공 후 워크스페이스를 선택하고 snapshot 결정을 준비하는지 테스트합니다.
  • 서버 snapshot이 비어 있고 로컬 데이터가 있으면 업로드 액션이 우선 표시되는지 테스트합니다.
  • 서버와 로컬 데이터가 모두 있으면 비교 요약과 3가지 선택지가 표시되는지 테스트합니다.
  • 401/403 오류가 auth-expired 상태로 분류되는지 테스트합니다.
  • 네트워크 실패 시 localStorage 저장이 유지되는지 테스트합니다.
  • Pages build에서 API URL 주입이 유지되는지 확인합니다.

구현 범위

  • 동기화 상태 계산 helper 추가
  • 로컬/서버 snapshot 비교 summary helper 추가
  • 로컬 모드 제약 warning component 추가
  • 데이터 관리 영역의 계정/동기화 UI 재구성
  • 서버 요청 실패와 세션 만료 상태 문구 정리
  • 공유 관리 섹션은 현재 기능을 유지하되 계정/동기화 섹션과 시각적으로 분리

Self Review

  • 서비스화 직후 필요한 사용자 혼란 해소에 초점을 맞췄습니다.
  • 분석/예측 기능은 다음 단계로 분리해 범위를 과하게 키우지 않았습니다.
  • 데이터 손실 위험이 있는 자동 덮어쓰기를 명시적으로 피했습니다.
  • 로컬 모드는 유지하되, 제약과 백업 필요성을 명확히 드러내도록 했습니다.
  • Cloudflare Pages 이전 호환성을 해치지 않도록 provider-specific UI나 runtime 기능을 넣지 않았습니다.
