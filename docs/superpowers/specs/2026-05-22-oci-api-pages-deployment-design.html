OCI API 배포 및 GitHub Pages 연결 설계

목표는 현재 로컬/정적 앱을 유지하면서, 실제 OCI API와 PostgreSQL schema를 연결해 로그인과 서버 동기화를 공개 환경에서 동작시키는 것입니다.

현재 상태

  • 프론트엔드는 apps/web Next static export이며 GitHub Pages에 배포됩니다.
  • 백엔드는 apps/api Fastify API이며 Dockerfile과 production Compose 파일이 있습니다.
  • DB는 OCI PostgreSQL 인스턴스의 gamja_apps_dev 안에 서비스 schema lcm, 테스트 schema lcm_test가 준비되어 있습니다.
  • DB role과 실제 접속 정보는 중앙 credentials sheet에 저장되어 있으며, repo에는 커밋하지 않습니다.
  • GitHub Pages 프론트는 NEXT_PUBLIC_API_BASE_URL이 설정되면 서버 기능을 사용하고, 없으면 localStorage 모드로 남습니다.
  • 프론트는 나중에 Cloudflare Pages로 옮길 수 있도록 특정 호스팅 플랫폼에 강하게 결합하지 않습니다.

접근안

추천OCI VM API + 기존 Pages 유지

  • API만 OCI VM에 Docker Compose로 올립니다.
  • 프론트는 GitHub Pages를 유지하고 빌드 시 API base URL만 주입합니다.
  • 가장 적은 변경으로 로그인, 동기화, 공유 기능을 실제 서비스처럼 검증할 수 있습니다.

프론트와 API를 모두 OCI로 이전

  • origin과 CORS가 단순해지지만, Pages로 이미 잘 동작하는 정적 배포 장점을 잃습니다.
  • 현재 단계에서는 운영 복잡도만 늘어납니다.

서버리스 또는 외부 플랫폼 이전

  • 장기적으로는 운영 부담을 줄일 수 있지만, 이미 OCI DB와 VM 접근 경로를 준비한 현재 흐름과 맞지 않습니다.
  • 지금은 실제 서비스 검증보다 플랫폼 이전 작업이 커집니다.

선택 설계

1차 서비스화는 OCI VM에 API 컨테이너를 배포하고 GitHub Pages가 해당 API를 바라보는 구조로 진행합니다. 프론트의 정적 배포 모델은 유지하고, 서버 기능은 API base URL이 있을 때만 활성화합니다. 

Browser
  -> https://sanghyun-io.github.io/living-cost-manager/
  -> https://API_PUBLIC_ORIGIN
       -> OCI reverse proxy
       -> living-cost-manager API container
       -> OCI PostgreSQL gamja_apps_dev, schema=lcm

Cloudflare Pages 호환성

이번 배포 대상은 GitHub Pages이지만, 프론트는 Cloudflare Pages로 이전 가능한 정적 산출물 구조를 유지합니다. 즉, 호스팅 제공자별 차이는 배포 설정에만 두고 애플리케이션 코드는 동일하게 유지합니다.

  • apps/web/out 정적 export 산출물을 단일 배포 artifact로 취급합니다.
  • NEXT_PUBLIC_API_BASE_URL은 빌드 시점 public 환경 변수로만 주입합니다.
  • API origin은 프론트 origin과 독립적으로 유지합니다.
  • GitHub Pages의 /living-cost-manager base path는 설정으로만 관리하고, 컴포넌트 내부에 하드코딩하지 않습니다.
  • Cloudflare Pages 이전 시에는 base path 없이 루트 배포가 가능하도록 asset path와 라우팅을 Next 설정에서만 제어합니다.
  • Cloudflare Pages Functions, KV, D1 같은 Cloudflare 전용 런타임 기능은 이번 단계에서 사용하지 않습니다.
  • 나중에 Cloudflare로 옮겨도 API는 계속 OCI Fastify 서버를 바라볼 수 있게 CORS 허용 origin만 추가/변경합니다.

배포 구성

  • API 서버는 OCI VM에서 Docker Compose로 실행합니다.
  • Compose의 db 서비스는 운영 연결에서는 사용하지 않고, API가 기존 OCI PostgreSQL 인스턴스의 lcm schema를 사용합니다.
  • 컨테이너 시작 전 prisma migrate deploy를 실행합니다.
  • API 공개는 직접 포트 노출이 아니라 reverse proxy 뒤에 둡니다.
  • HTTPS가 준비되기 전에는 로그인/동기화 기능을 공개 Pages에 연결하지 않습니다.

환경 변수와 비밀값

비밀값은 Git, README, GitHub Actions 로그, 채팅 출력에 남기지 않습니다.

  • DATABASE_URL: 중앙 credentials sheet의 값을 OCI VM 환경 파일에만 저장합니다.
  • JWT_SECRET: 새로 생성해 OCI VM 환경 파일에만 저장합니다.
  • CORS_ORIGIN: https://sanghyun-io.github.io
  • NEXT_PUBLIC_API_BASE_URL: GitHub Pages workflow variable 또는 secret으로 설정합니다. 이 값은 public URL이므로 credential은 아니지만, 배포 환경 설정으로 관리합니다.

DB 마이그레이션

  1. OCI VM에서 API 이미지 또는 repo 작업 디렉터리를 준비합니다.
  2. DATABASE_URL이 schema=lcm을 가리키는지 확인합니다.
  3. prisma migrate deploy를 실행합니다.
  4. API health check와 실제 auth/register smoke test를 실행합니다.

테스트 schema lcm_test는 API 테스트 전용입니다. 운영 배포 migration이나 수동 smoke test에는 사용하지 않습니다.

Pages 연결

  • GitHub Pages workflow에 NEXT_PUBLIC_API_BASE_URL을 추가합니다.
  • 배포된 정적 앱에서 회원가입, 로그인, 워크스페이스 조회, snapshot 업로드/다운로드를 확인합니다.
  • API URL이 없거나 API 호출이 실패하면 기존 localStorage 모드를 계속 사용할 수 있어야 합니다.
  • Pages 전용 설정은 GitHub Actions와 next.config.mjs로 제한하고, 앱 코드에는 호스팅 provider 분기를 추가하지 않습니다.

검증

  • pnpm -r --if-present build
  • pnpm test 또는 DB URL이 필요한 API 테스트를 포함한 equivalent command
  • API 컨테이너 /health 응답 확인
  • OCI PostgreSQL lcm schema에 migration table과 앱 테이블이 생성되는지 확인
  • GitHub Pages 배포 완료 후 실제 URL에서 앱 로딩 확인
  • 정적 산출물 apps/web/out이 provider 독립적으로 생성되는지 확인
  • 브라우저에서 회원가입/로그인 후 snapshot sync 수동 확인

롤백

  • 프론트 롤백: Pages workflow의 NEXT_PUBLIC_API_BASE_URL을 제거하거나 이전 커밋으로 되돌리면 localStorage 모드로 돌아갑니다.
  • API 롤백: OCI Compose에서 이전 이미지 또는 이전 git commit으로 재배포합니다.
  • DB 롤백: Prisma migration은 되돌리기보다, 현재 단계에서는 새 schema이므로 필요한 경우 lcm schema를 백업 후 재생성하는 방식을 사용합니다.

Self Review

  • 실제 서비스화 목표와 직접 관련 없는 기능 고도화는 제외했습니다.
  • 비밀값 저장 위치와 출력 금지 원칙을 명확히 했습니다.
  • 기존 Pages 정적 배포와 localStorage fallback을 유지합니다.
  • 운영 schema lcm과 테스트 schema lcm_test를 혼동하지 않도록 명시했습니다.
  • HTTPS 전 공개 연결 금지와 롤백 경로를 포함했습니다.
  • 향후 Cloudflare Pages 이전을 위해 호스팅 provider별 차이를 배포 설정으로 제한했습니다.
