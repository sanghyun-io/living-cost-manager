계정 및 동기화 UX 구현 계획

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

Goal: 로컬 모드 제약을 명확히 경고하고, 서버 계정/동기화 상태를 상태 카드 중심으로 정리한다.

Architecture: 동기화 상태 계산과 snapshot 비교 요약은 apps/web/app/lib/syncStatus.ts로 분리한다. UI는 기존 page.tsx 데이터 관리 모달 안에서 상태 카드, 로컬 모드 경고, 동기화 결정 카드로 재구성한다.

Tech Stack: Next static export, React client component, TypeScript, Vitest.

파일 구조

  • Create: apps/web/app/lib/syncStatus.ts - 상태 계산, 표시 문구, snapshot 요약
  • Modify: apps/web/tests/smoke.test.ts - helper 테스트 추가
  • Modify: apps/web/app/page.tsx - 데이터 관리 모달 UI 재구성
  • Modify: apps/web/app/globals.css - 상태 카드, 로컬 경고, 비교 요약 스타일
