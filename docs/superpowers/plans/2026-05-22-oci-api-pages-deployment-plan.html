OCI API 배포 및 Pages 연결 구현 계획

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

Goal: OCI VM의 기존 API gateway에 Living Cost Manager API를 배포하고, GitHub Pages 프론트가 공개 API를 사용해 로그인과 snapshot 동기화를 수행하게 만든다.

Architecture: 공개 API는 https://api.gamja.top/living-cost-manager/v1로 제공한다. Fastify는 같은 prefix 아래에 route를 등록하고, nginx는 해당 path를 OCI VM 내부 API 컨테이너 포트 4104로 proxy한다. 프론트는 정적 export를 유지하고 NEXT_PUBLIC_API_BASE_URL만 빌드 시점에 주입한다.

Tech Stack: Next static export, GitHub Pages, Fastify, Prisma, PostgreSQL, Docker Compose, nginx, OCI VM, pnpm, Vitest.

Step 3: env schema 확장

apps/api/src/env.ts의 schema에 API_BASE_PATH를 추가한다.

export const envSchema = z.object({
  NODE_ENV: z.enum(["development", "test", "production"]).default("development"),
  PORT: z.coerce.number().int().positive().default(4000),
  DATABASE_URL: z.string().min(1),
  JWT_SECRET: z.string().min(32),
  CORS_ORIGIN: z.url().default("https://sanghyun-io.github.io"),
  API_BASE_PATH: z
    .string()
    .trim()
    .regex(/^$|^\/[A-Za-z0-9/_-]+$/, "API_BASE_PATH must be empty or start with /")
    .transform((value) => value.replace(/\/+$/, ""))
    .default("")
});

Step 4: app route 등록 prefix-aware 변경

apps/api/src/app.ts에서 route 등록부를 plugin으로 감싼다.

const registerApiRoutes = async (api: typeof app) => {
  await api.register(authRoutes);
  await api.register(workspaceRoutes);
  await api.register(invitationRoutes);
  await api.register(memberRoutes);
  await api.register(snapshotRoutes);
  api.get("/health", async () => ({ ok: true }));
};

if (env.API_BASE_PATH) {
  await app.register(registerApiRoutes, { prefix: env.API_BASE_PATH });
} else {
  await registerApiRoutes(app);
}

기존 route 등록과 root /health 등록은 위 블록으로 대체한다.

services:

      context: .

OCI의 기존 PostgreSQL 인스턴스를 사용할 때는 `docker-compose.oci.yml`을 사용합니다. 이 Compose 파일은 API 컨테이너만 실행하고, `.env.oci`에서 `DATABASE_URL`과 `JWT_SECRET`을 읽습니다. `.env.oci`는 절대 커밋하지 않습니다.

Step 4: nginx proxy 추가

/etc/nginx/sites-enabled/api-gateway의 HTTPS server block에 아래 location을 추가한다.

# Living Cost Manager API (port 4104)
location ^~ /living-cost-manager/v1/ {
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Connection "";
    proxy_pass http://127.0.0.1:4104;
}

Step 5: nginx 검증과 공개 health 확인

ssh sh-oracle-cloud-dev-1 "sudo systemctl reload nginx"

Expected: 200 with {"ok":true}
