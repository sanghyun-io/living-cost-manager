Living Cost Manager OCI API 배포 Runbook

이 문서는 절차만 기록합니다. 실제 DATABASE_URL, JWT_SECRET, DB 비밀번호, 토큰, SSH key 값은 문서, Git commit, 터미널 출력, CI 로그에 남기지 않습니다.

대상

  • API VM: sh-oracle-cloud-dev-1
  • API 작업 디렉터리: /home/ubuntu/living-cost-manager
  • 내부 API 포트: 127.0.0.1:4104
  • 공개 API base URL: https://api.gamja.top/living-cost-manager/v1
  • DB: OCI PostgreSQL gamja_apps_dev, Prisma schema lcm

1. Repo 준비

ssh sh-oracle-cloud-dev-1 "mkdir -p /home/ubuntu/living-cost-manager"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && git init"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && git remote remove origin 2>/dev/null || true"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && git remote add origin https://github.com/sanghyun-io/living-cost-manager.git"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && git fetch origin main && git checkout -B main origin/main"

2. 환경 파일 생성

중앙 credentials sheet에서 living-cost-manager의 DATABASE_URL 값을 가져옵니다. 값은 로컬 셸 변수 또는 무시된 임시 파일 안에서만 다룹니다.

VM의 .env.oci에는 아래 key만 저장합니다.

DATABASE_URL=<central credentials sheet value>
JWT_SECRET=<generated secret>
ssh sh-oracle-cloud-dev-1 "chmod 600 /home/ubuntu/living-cost-manager/.env.oci"

3. Build, migration, start

ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && docker compose -f docker-compose.oci.yml build api"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && docker compose -f docker-compose.oci.yml run --rm api ./node_modules/.bin/prisma migrate deploy"
ssh sh-oracle-cloud-dev-1 "cd /home/ubuntu/living-cost-manager && docker compose -f docker-compose.oci.yml up -d api"

4. 내부 health 확인

ssh sh-oracle-cloud-dev-1 "curl -fsS http://127.0.0.1:4104/living-cost-manager/v1/health"

기대 결과:

{"ok":true}

5. nginx gateway 설정

/etc/nginx/sites-enabled/api-gateway의 HTTPS server block에 location을 추가합니다.

# Living Cost Manager API (port 4104)
location ^~ /living-cost-manager/v1/ {
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Connection "";
    proxy_pass http://127.0.0.1:4104;
}
ssh sh-oracle-cloud-dev-1 "sudo nginx -t"
ssh sh-oracle-cloud-dev-1 "sudo systemctl reload nginx"

6. 공개 health 확인

Invoke-WebRequest https://api.gamja.top/living-cost-manager/v1/health -UseBasicParsing

기대 결과: HTTP 200과 {"ok":true}

7. Pages 연결 확인

  1. GitHub repository variable NEXT_PUBLIC_API_BASE_URL이 공개 API base URL인지 확인합니다.
  2. Pages workflow가 성공했는지 확인합니다.
  3. https://sanghyun-io.github.io/living-cost-manager/에서 회원가입 또는 로그인을 수행합니다.
  4. 서버 session 생성, workspace 조회, snapshot 업로드/다운로드가 되는지 확인합니다.

Rollback

  • 프론트 롤백: GitHub variable NEXT_PUBLIC_API_BASE_URL 제거 후 Pages를 재배포합니다.
  • API 롤백: VM에서 이전 git commit으로 checkout 후 Compose를 다시 실행합니다.
  • nginx 롤백: /living-cost-manager/v1/ location을 제거하고 reload합니다.
